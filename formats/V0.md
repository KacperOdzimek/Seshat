# Format Version 0

## Fundamental

This apply to entire file:
```
Entry Names:    ASCII
Endian:         Little Endian
Float64:        IEEE 754 binary64
Compression:    Zstd 1.3.0 or greater
Magic:          SESHAT!?
```

## Notes

* Null termination on text is not required, as byte length is known

## Header

Valid seshat file begins with header:

```
Position:       Begin of file
Align:          8 bytes
```
```
uint8  magic[8]     // Magic sequence for correctness
uint64 version      // Format version
uint32 index        // Index bytes
uint32 names        // Names bytes
uint64 content      // Content bytes
```

## Index

### Indices Array

Index is array of entries, after the header.
Index array length = ``header:index`` / ``24 (entry size)``.

```
Position:       sizeof(header) // Already aligned 8 bytes
Align:          8 bytes
Entry align:    4 bytes
```
```
uint32 name offset  // Bytes offset from names array begin
uint32 data type    // Data type enumerator
uint64 data bytes   // Data length in bytes
uint64 data offset  // Data offset from content begin
```

### Names Array

Names array is directly after indices array. 

```
Position:       sizeof(header) + header:index_bytes
Align:          4 bytes
Entry align:    2 bytes
```
```
uint8 length
uint8 ascii[length]
```

## Contents

Contents are chunks of data linked to index fields.
Each chunk is guaranteed to keep 8 bytes align.
Chunks are guaranteed not to overlap.

```
Position:       sizeof(header) + header:index_bytes + header:names_bytes
Align:          8 bytes
Entry align:    8 bytes
```

## Data Types

Data type informs interpreter, what data is linked to entry.
Following data types exists:

### 0: Archive
A nested seshat file:

```
uint64 bytes    // Archive length
(Nested file)
```

### 1: Int64

```
int64 value     // Single int64 value
```

### 2: Float64
```
float64 value    // Single float64 value
```

### 3: UTF-8 Text
```
[Block of text - length entry:data_bytes]
```

### 4: Compressed UTF-8 Text
```
uint64 uncompressed size bytes
[Block of compressed text - length entry:data_bytes - 8 bytes]
```

### 5: Binary
```
[Block of binary data - length entry:data_bytes]
```

### 6: Compressed Binary
```
uint64 uncompressed size bytes
[Block of compressed binary data - length entry:data_bytes - 8 bytes]
```
